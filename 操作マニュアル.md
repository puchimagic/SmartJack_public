### 1.2 SmartJackとは

#### 1.2.1 アプリの概要

SmartJackは、本体と連携するスマートホーム管理アプリケーションです。お手持ちの赤外線リモコンの信号を学習し、スマートフォンから家電を操作できます。

#### 1.2.2 主な特徴

| 特徴 | 説明 |
|:----|:----|
| **階層的な管理** | 「家」→「部屋」→「リモコン」→「ボタン」の階層で直感的に管理 |
| **カスタムリモコン作成** | ボタンを自由にレイアウトしたオリジナルリモコンを作成 |
| **赤外線学習機能** | 既存のリモコン信号を自動的に学習・登録 |
| **シーンによる自動化** | スケジュールをトリガーに自動操作、複数操作をパッケージ化 |
| **UIのカスタマイズ** | テーマカラー、ライト/ダークモード、利き手設定に対応 |

#### 1.2.3 赤外線通信の特性と制限

赤外線（IR）通信には以下の特性があります。ご使用前にご理解ください。

| 特性 | 説明 |
|:----|:----|
| **直進性** | 赤外線は直進するため、本体と家電の間に障害物があると届きません |
| **壁を透過しない** | 赤外線は壁やドアを透過しません。同じ部屋内に設置してください |
| **距離の制限** | 送信距離は約3〜5mです（環境により異なります） |
| **向きが重要** | 本体のIR送信部を家電の受光部に向ける必要があります |
| **外光の影響** | 直射日光や強い照明の下では受信感度が低下することがあります |

```
× 壁越しの操作は不可
┌─────────┐    壁    ┌─────────┐
│ M5Atom  │ ──×──│  家電   │
└─────────┘    ║    └─────────┘
               ║
○ 同じ部屋内で直線上に配置
┌─────────┐         ┌─────────┐
│ M5Atom  │ ──────▶│  家電   │
└─────────┘   IR    └─────────┘
```

### 1.3 動作環境

#### 1.3.1 対応OS

| プラットフォーム | 対応バージョン |
|:---------------|:-------------|
| **iOS** | iOS 14.0 以上 |
| **Android** | Android 8.0 (API 26) 以上 |

#### 1.3.2 必要な権限

| 権限 | 用途 | 必須 |
|:----|:----|:----:|
| **Bluetooth** | 本体との通信 | ✅ |
| **位置情報** | BLE接続（Android） | △ |
| **通知** | スケジュール実行の通知 | △ |
| **Wi-Fi** | Wi-Fiモードでの通信 | △ |

> ⚠️ **注意**: Bluetooth権限は必須です。許可しない場合、本体と接続できません。

#### 1.3.3 ハードウェア要件

| 機器 | 説明 |
|:----|:----|
| **本体** | 赤外線学習・送信用デバイス（必須） |
| **IRユニット** | 赤外線送受信モジュール（本体に接続） |

#### 1.3.4 ネットワーク要件（Wi-Fiモード使用時）

| 項目 | 要件 |
|:----|:----|
| **Wi-Fi周波数帯** | **2.4GHz帯のみ対応**（5GHz帯は非対応） |
| **セキュリティ** | WPA/WPA2-PSK推奨 |
| **その他** | 本体とスマートフォンが同一ネットワーク内にあること |

> ⚠️ **重要**: 本体は**2.4GHz帯のWi-Fiのみ**に対応しています。5GHz帯のWi-Fiには接続できません。お使いのルーターが2.4GHz帯と5GHz帯を分けて出力しているか、または統合されているかをご確認ください。統合されている場合は、2.4GHz専用のSSIDを作成することをお勧めします。

### 1.4 用語説明

#### 1.4.1 基本用語

| 用語 | 説明 |
|:----|:----|
| **家** | 管理単位の最上位。「自宅」「実家」など複数の家を登録可能 |
| **部屋** | 家の中の空間単位。「リビング」「寝室」など |
| **リモコン** | 仮想リモコン。複数のボタンを配置できるキャンバス |
| **ボタン** | リモコン上の操作ボタン。タップでIR信号を送信 |
| **シーン** | 自動化ルール。トリガー条件と実行操作のセット |
| **トリガー** | シーンを発動させる条件（時刻など） |

#### 1.4.2 技術用語

| 用語 | 正式名称 | 説明 |
|:----|:--------|:----|
| **IR** | Infrared | 赤外線。家電リモコンの通信方式 |
| **BLE** | Bluetooth Low Energy | 省電力のBluetooth通信規格 |
| **本体** | - | ESP32-S3搭載の小型マイコンボード |
| **FAB** | Floating Action Button | 画面上に浮かぶ丸いアクションボタン |
| **AppBar** | Application Bar | 画面上部のタイトルバー領域 |

#### 1.4.3 画面・UI用語

| 用語 | 説明 |
|:----|:----|
| **ホーム画面** | アプリ起動時に表示されるメイン画面 |
| **ボトムナビゲーション** | 画面下部のタブ切り替えバー |
| **サイドメニュー（ドロワー）** | 画面左端からスワイプで表示されるメニュー |
| **編集モード** | ボタンの追加・移動・削除ができる状態 |
| **通常モード** | ボタンタップでIR送信できる状態 |

## 2. 基本操作

### 2.1 アカウント設定

SmartJackを利用するにはアカウントが必要です。アカウントにより、外出先からの操作やスケジュール実行が可能になります。

#### 2.1.1 サインアップ（新規アカウント作成）

**操作手順**:
1. アプリを起動
2. ログイン画面が表示される
3. 「アカウントを作成」をタップ
4. メールアドレスを入力
5. パスワードを設定（8文字以上、大文字・小文字・数字を含む）
6. 「アカウント作成」をタップ
7. 確認コードがメールで届く
8. 確認コードを入力
9. アカウント作成完了

```
┌─────────────────────────────┐
│       SmartJack             │
├─────────────────────────────┤
│                             │
│ メールアドレス              │
│ ┌─────────────────────────┐│
│ │ example@email.com        ││
│ └─────────────────────────┘│
│                             │
│ パスワード                    │
│ ┌─────────────────────────┐│
│ │ ••••••••                ││
│ └─────────────────────────┘│
│                             │
│     [ログイン]                │
│                             │
│   アカウントを作成            │
└─────────────────────────────┘
```

#### 2.1.2 サインイン

**操作手順**:
1. アプリを起動
2. ログイン画面が表示される
3. メールアドレスを入力
4. パスワードを入力
5. 「ログイン」をタップ
6. ホーム画面が表示される

#### 2.1.3 パスワードを忘れた場合

**操作手順**:
1. ログイン画面で「パスワードを忘れた場合」をタップ
2. メールアドレスを入力
3. 「リセットリンクを送信」をタップ
4. メールに届いたリンクから新しいパスワードを設定

#### 2.1.4 サインアウト

**操作手順**:
1. 設定画面を開く
2. 「サインアウト」をタップ
3. 確認ダイアログで「サインアウト」をタップ
4. ログイン画面に戻る

> ⚠️ **注意**: サインアウトしてもローカルデータ（リモコン、ボタン等）は削除されません。

### 2.7 リモコン操作（IR送信）

#### 2.7.1 通常モードでのボタン操作

リモコン操作画面でボタンをタップすると、IR信号が送信されます。

**操作手順**:
1. ホーム画面でリモコンを選択
2. リモコン操作画面が表示される
3. 操作したいボタンをタップ
4. 本体経由でIR信号が送信される
5. 対象の家電が操作される

```
┌─────────────────────┐
│ ← テレビA      [✏️] │ ← ✏️は編集モード切替
├─────────────────────┤
│                     │
│   ┌─────┐          │
│   │ 電源 │ ←タップ  │ → IR送信 → 📺テレビON/OFF
│   └─────┘          │
│                     │
│  ┌───┐ ┌───┐ ┌───┐│
│  │ch+│ │音+│ │入力││
│  └───┘ └───┘ └───┘│
│  ┌───┐ ┌───┐ ┌───┐│
│  │ch-│ │音-│ │消音││
│  └───┘ └───┘ └───┘│
└─────────────────────┘
```

#### 2.7.2 操作の流れ

```
[ボタンタップ]
     │
     ▼
[アプリがIRコードを取得]
     │
     ▼
[BLE/Wi-FiでM5に送信]
     │
     ▼
[本体がIR信号を発信]
     │
     ▼
[家電がIR信号を受信]
     │
     ▼
[家電が操作される]
```

> ⚠️ **注意**: 本体が接続されていない場合、IR送信は失敗します。事前に接続状態を確認してください。

### 2.8 編集モードと通常モード

リモコン操作画面には2つのモードがあります。

#### 2.8.1 モードの違い

| モード | ボタンタップ時の動作 | 用途 |
|:------|:-------------------|:----|
| **通常モード** | IR信号を送信 | 日常的なリモコン操作 |
| **編集モード** | ボタンの編集ダイアログを表示 | ボタンの追加・編集・削除・移動 |

#### 2.8.2 モードの切り替え

**操作手順**:
1. リモコン操作画面を開く
2. AppBar右側の編集アイコン（✏️）をタップ
3. モードが切り替わる

```
通常モード                        編集モード
┌─────────────────────┐          ┌─────────────────────┐
│ ← テレビA      [✏️] │  ─────▶  │ ← テレビA  [+][🗑️][✓]│
├─────────────────────┤          ├─────────────────────┤
│ ボタンタップ→IR送信 │          │ ボタンタップ→編集   │
│                     │          │ ドラッグ→位置移動   │
└─────────────────────┘          └─────────────────────┘
       ✏️タップ                        ✓タップ
       ◀─────────────────────────────────────
```

#### 2.8.3 編集モードでできること

| 操作 | 方法 |
|:----|:----|
| ボタンの追加 | AppBarの「+」をタップ |
| ボタンの編集 | ボタンをタップ |
| ボタンの削除 | ボタンを選択 → 🗑️をタップ |
| ボタンの移動 | ボタンをドラッグ |
| 編集モード終了 | ✓をタップ |

**クラウド登録のメリット**:

| 機能 | ローカルのみ | クラウド登録後 |
|:----|:-----------|:-------------|
| 同じWi-Fi内での操作 | ✅ | ✅ |
| 外出先からの操作 | ❌ | ✅ |

> ⚠️ **注意**: クラウド登録には、本体がWi-Fi接続済みであることが必要です。

**登録状態の確認**:
- 部屋一覧で☁️アイコンが表示されていれば登録済み

### 3.3 リモコンの管理

4. リモコン名を入力（例：「テレビA」「エアコン」）

#### 3.3.5 リモコンの並べ替え

リモコンを特定の部屋に割り当てると、ホーム画面でその部屋を選択したときにリモコンが表示されます。

**検索のコツ**:

### 3.4 ボタンの管理とIR学習

ボタンはリモコン上に配置する操作ボタンです。既存のリモコンから赤外線信号を学習して登録できます。

#### 3.4.1 リモコン操作画面へのアクセス

**操作手順**:
1. ホーム画面で部屋を選択
2. リモコングリッドから目的のリモコンをタップ
3. リモコン操作画面が表示される

または

2. 「開く」をタップ

#### 3.4.2 編集モードへの切り替え

ボタンの追加・編集・削除には編集モードへの切り替えが必要です。

### 3.8 デバイス接続（本体）

SmartJackは本体を経由して赤外線信号の学習・送信を行います。

#### 3.8.1 本体について

本体は、ESP32-S3を搭載した小型マイコンボードです。IRユニットを接続することで、赤外線の送受信が可能になります。

**ハードウェア仕様**:

| コンポーネント | 説明 |
|:-------------|:----|
| **RGB LED** | 動作状態を色で表示 |
| **ユーザーボタン** | 長押しでBLE/Wi-Fiモード切り替え |
| **IR受信部** | 赤外線信号の学習用 |
| **IR送信部** | 赤外線信号の送信用 |

#### 3.8.2 LEDインジケーター

本体のLED色で現在の状態を確認できます。

| LED色 | パターン | 意味 |
|:-----|:--------|:-----|
| 🔴 赤 | 常時点灯 | BLE待機中（未接続） |
| 🟢 緑 | 常時点灯 | BLE接続中 |
| 🟡 黄 | 点滅 | Wi-Fi接続試行中 |
| 🔵 青 | 常時点灯 | Wi-Fi接続済み |
| 🟣 紫 | 点滅 | IR学習中（信号待機中） |
| 🔴 赤 | 1秒点滅 | IR送信完了 |

#### 3.8.3 BLE接続

Bluetooth Low Energyでアプリとデバイスを接続します。

**操作手順**:
1. 本体の電源をON
2. LEDが🔴赤色に点灯（BLE待機中）
3. アプリでBLE接続を開始
4. デバイス一覧から本体を選択
5. 接続が完了するとLEDが🟢緑色に点灯

**接続できない場合**:
- 本体が起動しているか確認
- スマートフォンのBluetoothがONになっているか確認
- アプリのBluetooth権限が許可されているか確認
- 本体を再起動

#### 3.8.4 Wi-Fi設定

Wi-Fiモードを使用すると、BLEより広い範囲でデバイスと通信できます。

> ⚠️ **重要: 2.4GHz帯Wi-Fiのみ対応**
>
> 本体は**2.4GHz帯のWi-Fiのみ**に対応しています。5GHz帯には接続できません。
>
> **2.4GHz帯と5GHz帯の見分け方**:
> - ルーターの設定画面でSSIDを確認
> - 一般的に「xxx-5G」「xxx_5GHz」は5GHz帯、「xxx-2G」「xxx_2.4GHz」は2.4GHz帯
> - 統合SSID（バンドステアリング）の場合は、2.4GHz専用のSSIDを作成することを推奨

> ⚠️ **ルーター設定の確認事項**
>
> 以下の設定が有効になっていると、アプリから本体を検出・操作できない場合があります。
>
> | 設定項目 | 説明 | 必要な対処 |
> |:--------|:----|:----------|
> | **プライバシーセパレーター**<br>（APアイソレーション） | 同じWi-Fi内のデバイス間通信をブロックする機能 | **OFF**にする |
> | **MACアドレスフィルタリング** | 許可されたデバイスのみ接続可能にする機能 | 本体のMACアドレスを許可リストに追加 |
>
> ※ 設定方法はルーターの機種により異なります。取扱説明書をご確認ください。

> ⚠️ **スマートフォン側の確認事項（mDNS関連）**
>
> Wi-Fiモードでは**mDNS**（マルチキャストDNS）を使用してデバイスを検出します。以下の設定が有効になっていると、デバイスを検出できない場合があります。
>
> | 項目 | 影響 | 対処法 |
> |:----|:----|:------|
> | **VPNアプリ** | VPN接続中はmDNSマルチキャストがブロックされる | VPNをOFFにする |
> | **プライベートDNS**（Android） | mDNS解決に影響する場合がある | 設定 → ネットワーク → プライベートDNS → OFF |
>
> 💡 デバイスが見つからない場合は、まずVPNアプリをOFFにしてお試しください。

**操作手順**:
1. BLEで接続した状態で設定を開始
2. Wi-Fi設定画面を開く
3. SSIDを入力（**2.4GHz帯**のWi-Fiの名前）
4. パスワードを入力
5. ホスト名を入力（任意）
6. 「設定」をタップ
7. 本体がWi-Fi接続を試行（LED🟡黄色点滅）
8. 接続成功でLED🔵青色点灯

#### 3.8.5 通信モードの切り替え

本体は、BLEモードとWi-Fiモードを切り替えることができます。

**切り替え方法**:
1. 本体のユーザーボタンを長押し（1秒以上）
2. モードが切り替わる
3. LEDの色で現在のモードを確認

| モード | LED色 | 特徴 |
|:------|:-----|:----|
| **BLEモード** | 🔴赤/🟢緑 | 近距離通信、省電力 |
| **Wi-Fiモード** | 🔵青 | 広範囲通信、mDNS対応 |

## 4. トラブルシューティング

### 4.1 接続に関する問題

#### 4.1.1 本体が見つからない（BLE）

**症状**: デバイス一覧に本体が表示されない

**原因と対処法**:

| 原因 | 対処法 |
|:----|:------|
| 本体の電源が入っていない | 電源をONにする。LEDが点灯することを確認 |
| 本体がWi-Fiモードになっている | ユーザーボタンを長押ししてBLEモードに切り替え |
| スマートフォンのBluetoothがOFF | 設定からBluetoothをONにする |
| アプリのBluetooth権限が未許可 | 端末の設定からSmartJackのBluetooth権限を許可 |
| 本体が他のデバイスに接続中 | 他のデバイスとの接続を切断 |
| 本体との距離が遠い | 3m以内に近づく |

**確認手順**:
1. 本体のLEDが🔴赤色に点灯しているか確認
2. スマートフォンのBluetooth設定を確認
3. アプリを完全に終了して再起動
4. 本体を再起動（電源OFF → ON）

#### 4.1.2 BLE接続が切れる

**症状**: 接続してもすぐに切断される、または不安定

**原因と対処法**:

| 原因 | 対処法 |
|:----|:------|
| 距離が遠すぎる | 本体に近づく（推奨：3m以内） |
| 障害物がある | 間にある障害物（壁、金属など）を避ける |
| 電波干渉 | Wi-Fiルーターや電子レンジから離れる |
| バッテリー残量低下 | 本体の電源を確認 |
| アプリのバックグラウンド制限 | 端末設定でバックグラウンド実行を許可 |

#### 4.1.3 Wi-Fi接続ができない

**症状**: Wi-Fi設定後、接続に失敗する（LEDが🟡黄色のまま）

**原因と対処法**:

| 原因 | 対処法 |
|:----|:------|
| SSIDまたはパスワードが間違っている | 正確なSSID・パスワードを再入力 |
| 2.4GHz帯でない | 本体は2.4GHzのみ対応。5GHz帯は使用不可 |
| ルーターの接続台数制限 | 他のデバイスを切断して空きを作る |
| ルーターの設定（MACフィルタリング等） | ルーターの設定を確認 |
| プライバシーセパレーターがON | ルーター設定でOFFにする（3.8.4参照） |
| Wi-Fiの電波が弱い | ルーターに近づける |

**確認手順**:
1. スマートフォンで同じWi-Fiに接続できるか確認
2. Wi-Fiが2.4GHz帯であることを確認
3. SSID・パスワードに特殊文字が含まれていないか確認
4. ルーターのセパレーター機能を確認（3.8.4参照）
5. 本体を再起動してやり直す

#### 4.1.4 Wi-Fi接続後にアプリから操作できない

**症状**: LEDは🔵青色だが、アプリからIR送信できない

**原因と対処法**:

| 原因 | 対処法 |
|:----|:------|
| スマートフォンが別のネットワークに接続 | 同じWi-Fiネットワークに接続 |
| ホスト名が解決できない（mDNS） | 下記「mDNS関連の確認事項」参照 |
| プライバシーセパレーターがON | ルーター設定でOFFにする（3.8.4参照） |
| ファイアウォールの設定 | ルーターの設定を確認 |

> 💡 **mDNS関連の確認事項**
>
> Wi-Fiモードでデバイスが見つからない場合、以下を確認してください：
>
> 1. **VPNアプリをOFFにする** - VPN接続中はmDNSが機能しません
> 2. **Androidの場合**:
>    - 設定 → ネットワーク → プライベートDNS → 「OFF」
> 3. **ルーターのセパレーター機能をOFFにする**（3.8.4参照）
>
> これらを確認しても解決しない場合は、BLEモードでの接続をお試しください。

### 4.2 IR学習に関する問題

#### 4.2.1 IR学習が成功しない

**症状**: 「学習開始」をタップしても信号を受信できない

**原因と対処法**:

| 原因 | 対処法 |
|:----|:------|
| リモコンの向きが合っていない | リモコンのIR発光部をM5のIR受信部に向ける |
| 距離が遠すぎる | 5〜10cmまで近づける |
| リモコンの電池切れ | リモコンの電池を交換 |
| 学習タイミングがずれている | LEDが🟣紫色に点滅してからボタンを押す |
| 直射日光・蛍光灯の干渉 | 日光を避け、蛍光灯から離れる |
| 非対応のリモコン | 赤外線方式でないリモコンは学習不可（RF、Bluetooth等） |

**IR学習のコツ**:

```
         リモコン
            │
       IR発光部 → ─────────→ ← IR受信部
            │                    │
            │      5〜10cm       │
            └────────────────────┘
                 本体
```

1. LEDが🟣紫色に点滅するまで待つ
2. リモコンを本体に向けて5〜10cmまで近づける
3. リモコンのボタンを**短く1回**押す（長押ししない）
4. LEDの色が変わったら学習成功

#### 4.2.2 学習したIRコードが正しく動作しない

**症状**: 学習は成功するが、送信しても家電が反応しない

**原因と対処法**:

| 原因 | 対処法 |
|:----|:------|
| 学習時のノイズ混入 | 静かな環境で再学習 |
| 複数回の信号が混入 | ボタンを短く1回だけ押して再学習 |
| 送信時の距離・向き | 本体の向きを家電に合わせる |

**対処手順**:
1. 編集モードでボタンを選択
2. 「再学習」をタップ
3. 環境を整えて再度学習
4. テスト送信で動作確認

### 4.3 IR送信に関する問題

#### 4.3.1 ボタンをタップしても家電が反応しない

**症状**: ボタンをタップしても何も起きない

**赤外線の基本を確認**:

赤外線は光の一種であり、以下の特性があります。

| 特性 | 影響 |
|:----|:----|
| 壁を透過しない | 別の部屋からは操作不可 |
| 障害物に遮られる | 家具や人が間にあると届かない |
| 直進する | 向きが合っていないと届かない |
| 距離に限界がある | 3〜5m以上離れると届きにくい |

**チェックリスト**:

| 確認項目 | 対処法 |
|:--------|:------|
| 本体と接続されているか | LEDが🟢緑または🔵青であることを確認 |
| 編集モードになっていないか | 通常モードに切り替える（✓をタップ） |
| IRコードが登録されているか | 編集モードでボタンを確認、必要に応じて再学習 |
| 本体の向き | IR送信部を家電に向ける |
| 距離が遠すぎないか | 3〜5m以内に設置 |
| 障害物がないか | 本体と家電の間に障害物を置かない |
| 同じ部屋にあるか | **赤外線は壁を透過しません** |
| 家電の受光部が見えるか | 家電のリモコン受光部に向ける |

**送信時のLED確認**:
- 送信成功時：LED🔴赤色が1秒点滅
- 点滅しない場合：接続が切れている可能性

#### 4.3.2 特定のボタンだけ動作しない

**症状**: 一部のボタンのみ反応しない

**対処法**:
1. 該当ボタンを編集モードで開く
2. IRコードが登録されているか確認
3. 「再学習」で再度学習
4. テスト送信で確認

#### 4.3.3 エアコンの一部機能が動作しない

**症状**: 電源は入るが、温度変更やモード切替が効かない

**原因と対処法**:

| 原因 | 対処法 |
|:----|:------|
| エアコンのIR信号は複合的 | 各機能（温度、モード等）を個別に学習 |
| サイクルボタンの順序ずれ | 現在の状態と同期を取ってからタップ |
| 状態を含む信号の学習 | エアコンの状態を固定して学習し直す |

> 💡 **ヒント**: エアコンのリモコンは状態（温度、モード、風量など）をまとめて送信することが多いです。それぞれの状態ごとに別々のボタンとして学習すると、より確実に動作します。

### 4.4 シーンに関する問題

#### 4.4.1 スケジュールシーンが実行されない

**症状**: 設定した時刻になっても自動実行されない

**原因と対処法**:

| 原因 | 対処法 |
|:----|:------|
| シーンが無効（OFF）になっている | シーン画面でスイッチをONにする |
| アプリが終了している | アプリをバックグラウンドで起動したままにする |
| 通知権限が未許可 | 端末設定で通知を許可 |
| バッテリー最適化が有効 | SmartJackをバッテリー最適化の対象外に設定 |
| 曜日設定が間違っている | シーン編集で実行曜日を確認 |
| 認証トークンが期限切れ | アプリを開いて再サインイン |
| OSによる実行遅延 | **仕様です**（下記参照） |

> 💡 **OSの制限による実行時刻のずれについて**
>
> iOS/Androidでは、バッテリー消費を抑えるためにバックグラウンド処理のタイミングをOSが制御しています。これにより、**設定時刻から数分〜最大10分程度ずれて実行される**ことがあります。これはOSの仕様であり、完全に防ぐことはできません。
>
> **対策**:
> - 時間に余裕を持った設定にする（例：7:00起床なら6:50にエアコンON）
> - バッテリー最適化を無効にする（下記参照）
> - 時間厳守が必要な用途には使用しない

**Android向け対処**:
1. 設定 → アプリ → SmartJack → バッテリー → 「制限なし」を選択
2. 設定 → アプリ → SmartJack → 通知 → 有効にする

**iOS向け対処**:
1. 設定 → SmartJack → 通知 → 許可
2. 設定 → SmartJack → Appのバックグラウンド更新 → 有効

> ⚠️ **認証トークンの期限について**
>
> SmartJackはセキュリティ上の理由から、認証トークンの有効期限が**30日間**に設定されています。30日以上アプリを開かないと、スケジュール実行やMQTT通信による操作ができなくなります。
>
> **対処法**: アプリを開いて再度サインインしてください。定期的（月に1回程度）にアプリを開くことをお勧めします。

### 4.7 よくある質問（FAQ）

#### Q1: 対応している家電は何ですか？

**A**: 赤外線リモコンで操作できる家電であれば、基本的に対応しています。

| 対応 | 非対応 |
|:----|:------|
| テレビ、エアコン、照明、扇風機、オーディオ、DVDプレーヤーなど | Bluetooth機器、Wi-Fi直接操作機器、RFリモコン機器 |

#### Q2: 複数の本体を使えますか？

**A**: はい、部屋ごとに異なる本体を設定できます。広い家や複数の部屋に設置する場合に便利です。

#### Q3: 外出先から操作できますか？

**A**: はい、可能です。以下の条件を満たすと、アプリが自動的にMQTT通信に切り替わり、外出先からでも操作できます。

**必要な設定**:
1. デバイスをクラウド登録している（3.2.7参照）
2. デバイスがWi-Fiに接続されている
3. アプリにサインインしている

**動作の仕組み**:
- 自宅のWi-Fiに接続中 → ローカルWi-Fi通信（高速）
- 外出先（モバイル回線等）→ MQTT通信（インターネット経由）

> 💡 切り替えはアプリが自動で行うため、ユーザーが意識する必要はありません。

#### Q4: 家族で共有できますか？

**A**: 同じネットワーク内であれば、複数の端末から本体に接続して操作できます。ただし、同時接続には制限があります。

#### Q5: リモコンのボタン数に制限はありますか？

**A**: アプリ上の制限は特にありませんが、多数のボタンを登録すると画面が見づらくなる場合があります。機器ごとにリモコンを分けることをお勧めします。

#### Q6: 電池が必要ですか？

**A**: 本体はUSB-Cで電源供給します。電池は不要ですが、常時電源に接続しておく必要があります。
